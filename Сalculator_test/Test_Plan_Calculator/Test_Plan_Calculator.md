# Тест-план: Виртуальный калькулятор
Версия 1.0  
Дата: 04.02.2026

## 1. Введение
**1.1. Цель документа:** Определить стратегию, объём, график и критерии тестирования гипотетического продукта "Виртуальный калькулятор".
**1.2. Объект тестирования:** Веб-приложение "Калькулятор" с базовыми арифметическими функциями: сложение, вычитание, деление умножение.
**1.3. Цель тестирования:** Верифицировать корректность реализации бизнес-логики математических операций согласно самостоятельно определённым требованиям.

## 2. Объём тестирования
**Что будет тестироваться:**
*   Функциональность базовых операций: **сложение**, **вычитание**, **умножение**, **диление**.
*   Обработка различных классов входных данных: положительные, отрицательные, минимальные/максимальные, дробные числа.
*   Корректность вычисления результата.

**Что НЕ будет тестироваться :**
*   Пользовательский интерфейс (UI), так как продукт виртуальный.
*   Производительность, безопасность, совместимость.

## 3. Подходы и виды тестирования
*   **Функциональное тестирование:** Проверка соответствия операций математической логике на основе позитивных и негативных сценариев.
*   **Тестирование граничных значений:** Проверка обработки чисел с максимальной и минимальной длиной, а также операций с нулём.
*   **Тестирование классов эквивалентности:** Проверка работы с положительными и отрицательными числами.

## 4. Критерии начала и окончания тестирования
*   **Критерии начала:** Наличие самостоятельно определённого набора требований/ограничений.
*   **Критерии успешного окончания:**
    *   Выполнены 100% запланированных тест-кейсов с высоким приоритетом.
    *   Достигнуто целевое покрытие ключевых функций.

## 5. Распределение ролей и ответственности
*   **Тестировщик / Автор плана:** Сергей — разработка тест-плана, создание и выполнение тест-кейсов, документирование результатов.

## 6. График тестирования
*   **Планирование и дизайн тестов:** 01.11.2025 – 30.11.2025
*   **Выполнение функционального тестирования:** 01.12.2025 – 20.12.2025
## 7. Окружение, инструменты, артефакты
*   **Тестовая среда:** Не требуется (виртуальный продукт).
*   **Инструменты:** Система контроля версий Git, хостинг GitHub для хранения артефактов.
*   **Артефакты, которые будут созданы:**
    *   Тест-план.
    *   Набор тест-кейсов.
## 8. Риски и пути их минимизации
| Риск | Влияние | Вероятность | Способ снижения |
| :--- | :--- | :--- | :--- |
| Неполнота или противоречивость самостоятельно определённых "требований" | Среднее | Средняя | Чёткая фиксация всех допущений и ограничений в разделе 2 этого документа. |
| Пропуск важных негативных сценариев | Высокое | Низкая | Применение техник тест-дизайна (граничные значения, классы эквивалентности). |
| Ошибки в эталонных расчётах для сверки результатов | Критическое | Низкая | Использование стороннего проверенного инструмента (калькулятор ОС, Excel) для перекрёстной проверки. |
